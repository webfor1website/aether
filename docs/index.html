What is Aether

Aether is a provenance-first programming language and toolchain that tracks, scores, and governs AI-generated code. Every function carries provenance metadata about who (or what) wrote it, and Aether enforces trust policies based on this information.

Provenance Tracking

Every function is tagged with @prov(source: "...", confidence: ...) metadata that tracks the AI model, human author, or tool that created it.

Trust Scoring

Aether calculates weighted trust scores based on provenance, allowing you to set policies that balance automation with safety.

Policy Enforcement

Configure trust thresholds and enforcement modes (report vs enforce) to match your team's risk tolerance and workflow.

Think of Aether as an immune system for your codebase - it identifies potentially risky AI-generated code and helps you make informed decisions about what to run, ship, or modify.

Quick Start

Get started with Aether in three simple steps:

1. Initialize a new project

$ aether init myproject
[aether] created: main.ae
[aether] created: .aether-policy.toml
[aether] created: .aether-wellbeing
[aether] initialized new project: myproject
[aether] run: aether run --session-id dev main.ae

2. Run your first program

$ cd myproject
$ aether run --session-id dev main.ae
  final trust score: 1.00
Result: 42

3. Generate a trust report

$ aether report .
[aether] main.ae: trust score 1.00 (user provenance)

That's it! You now have a provenance-aware project with trust scoring. The .aether-policy.toml file lets you configure trust policies for your team.

CLI Reference

Aether provides a complete command-line interface for project management, code execution, and trust analysis.

init

Create a new Aether project with default configuration files.

$ aether init [project_name]
$ aether init  # Initialize in current directory

integrate

Onboard existing codebases by scanning for source files and generating policy configuration.

$ aether integrate
[aether] found 12 Rust files, 8 Python files, 3 TypeScript files
[aether] generated .aether-policy.toml

dev

Watch for file changes and automatically re-run programs with hot reloading.

$ aether dev
[aether] watching for changes... (Ctrl+C to stop)
[aether] file changed: main.ae — re-running...
  final trust score: 1.00
Result: 42

run

Execute Aether programs with trust enforcement and provenance tracking.

$ aether run --session-id dev main.ae
$ aether run --session-id dev --min-trust 0.8 main.ae
$ aether run --session-id dev --mode report main.ae

report

Generate trust reports for files or directories, with support for SARIF output.

$ aether report .
$ aether report --format sarif src/ > results.sarif

wrap

Generate Aether extern stubs for Rust, Python, or TypeScript files.

$ aether wrap mylib.rs --source claude --confidence 0.7
$ aether wrap mymodule.py --source ai --confidence 0.7
$ aether wrap mycomponent.ts --source ai --confidence 0.7

replay

Replay a previous session to review execution details and trust scores.

$ aether replay --session-id dev

check

Parse and type-check Aether files without execution.

$ aether check main.ae

Provenance tags

Provenance tags are the core of Aether's tracking system. Every function can be annotated with @prov metadata:

@prov(source: "user", confidence: 1.0)
fn main() -> Int {
  42
}

@prov(source: "claude", confidence: 0.85)
fn calculate_total(items: List) -> Float {
  // AI-generated implementation
}

@prov(source: "cursor", confidence: 0.75)
fn process_user_data(data: UserData) -> ProcessedData {
  // Cursor-generated implementation
}

Source values

  • "user" - Human-written code
  • "claude" - Generated by Claude
  • "cursor" - Generated by Cursor
  • "grok" - Generated by Grok
  • "ai" - Generic AI source

Confidence scores

Confidence ranges from 0.0 to 1.0, representing the certainty of the provenance attribution. Higher confidence means more reliable trust scoring.

Trust scoring

Aether calculates trust scores using a weighted formula that considers provenance, confidence, and function characteristics.

Weighted trust formula

trust_score = Σ(prov_confidence × source_weight × function_weight) / Σ(function_weight)

Source weights (default)

  • user: 1.0 (full trust)
  • claude: 0.85
  • cursor: 0.80
  • grok: 0.75
  • ai: 0.70

Function weights

  • Entry points (main): 1.0
  • Public functions: 0.9
  • Private functions: 0.7
  • Helper functions: 0.5

The weighted approach ensures that critical functions and human-written code have more influence on the overall trust score.

Policy config

The .aether-policy.toml file configures trust policies for your project:

[trust]
min_trust = 0.7
mode = "report"
scoring = "weighted"

[sources]
allowed = ["user", "claude", "cursor", "grok", "ai"]

[output]
format = "text"

Trust section

  • min_trust: Minimum trust score threshold (0.0-1.0)
  • mode: "report" (warn) or "enforce" (block)
  • scoring: "weighted" or "flat"

Sources section

Define which provenance sources are allowed in your codebase.

Output section

Configure output format for reports and messages.

HTTP API

Aether Server provides a REST API for trust analysis and health checks.

POST
/analyze

Analyze code and return trust scores and provenance information.

POST /analyze
Content-Type: application/json

{
  "code": "@prov(source: \"claude\", confidence: 0.85)\nfn main() -> Int { 42 }",
  "options": {
    "scoring": "weighted",
    "include_details": true
  }
}
GET
/health

Check server health and status.

GET /health

{
  "status": "healthy",
  "version": "0.1.0",
  "uptime": "2h 34m"
}

Running the server

$ aether-server --port 8080
[aether] server listening on :8080

CI Integration

Integrate Aether into your CI pipeline to enforce trust policies automatically.

GitHub Actions

name: Aether Trust Check
on: [push, pull_request]

jobs:
  aether-check:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Setup Aether
        run: |
          wget https://github.com/webfor1website/aether/releases/latest/download/aether-cli-linux-x64
          chmod +x aether-cli-linux-x64
          sudo mv aether-cli-linux-x64 /usr/local/bin/aether
      - name: Run trust analysis
        run: |
          aether report --format sarif . > results.sarif
      - name: Upload SARIF
        uses: github/codeql-action/upload-sarif@v2
        with:
          sarif_file: results.sarif

The CI integration will fail builds that don't meet your trust thresholds and provide detailed SARIF reports for GitHub code scanning.

Docker

Run Aether in Docker containers for consistent environments and easy deployment.

Dockerfile

FROM rust:1.70 as builder
WORKDIR /app
COPY . .
RUN cargo build --release --bin aether-cli

FROM debian:bookworm-slim
RUN apt-get update && apt-get install -y ca-certificates && rm -rf /var/lib/apt/lists/*
COPY --from=builder /app/target/release/aether-cli /usr/local/bin/aether
ENTRYPOINT ["aether"]

Usage

$ docker build -t aether .
$ docker run --rm -v $(pwd):/workspace aether run --session-id ci /workspace/main.ae

Interop

Aether can wrap existing code from multiple languages, generating extern stubs with provenance metadata.

Rust

$ aether wrap mylib.rs --source claude --confidence 0.7

# Output: mylib.aeth
@prov(source: "claude", confidence: 0.85)

extern fn calculate_total(Vec, Float) -> Float;
extern fn process_user_data(String, Int, Bool) -> String;

Python

$ aether wrap mymodule.py --source ai --confidence 0.7

# Output: mymodule.aeth
@prov(source: "claude", confidence: 0.85)

extern fn calculate_total(Int, Float) -> Float;
extern fn process_user_data(String, Int, Bool) -> String;

TypeScript

$ aether wrap mycomponent.ts --source ai --confidence 0.7

# Output: mycomponent.aeth
@prov(source: "claude", confidence: 0.85)

extern fn processUserData(String, Int, Bool) -> String;
extern fn calculateUserScore(Int, Int) -> Int;

Generated stubs can be imported and used in Aether programs:

import "mylib.aeth"

@prov(source: "user", confidence: 1.0)
fn main() -> Float {
  let items = vec![Item { name: "test", price: 10.0 }];
  calculate_total(items, 0.08)
}
